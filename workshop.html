XML Tags

  • Tags have names, and come in three flavours: start, end, empty
  • Tags start and end with less-than and greater-than characters.
  • Start tag: 
    <tagname>
  • End tag: 
    </tagname>
  • Empty tag: 
    <tagname/>

XML Tags

  • Names are case sensitive.
  • Names start with a letter.
  • Names may contain digits, hyphens, underscores, and colons.
  • Colons in names have a special purpose, so try to avoid them.

XML Tags

  • Well-formed: start tags must match up with an end tag.
  • Tags must be nested properly.

XML Tags

<title><b>Frankie & Johnny</b></title>

Nesting

Don't do this:


              Some text <i>in italic
              and <b>bold and</i> not italic</b>

Do do this instead:


              Some text <i>in italic and <b>bold
                and</b></i><b> not italic</b>

XML Attributes

Start and empty tags can have attributes. End tags do not.

Attributes have a name and a value.

<title part='main'></title>
<pagebreak n="27"/>
<title part='main' font='gothic'></title>

XML Attributes

<title part='main' font='gothic'></title>
  • Attribute names are case-sensitive.
  • Attribute names start with a letter
  • Attribute values must be quoted

XML Attributes

  • Attribute names may contain digits, hyphens, underscores, and colons.
  • Colons in attribute names have a special purpose, so try to avoid them.
  • Order doesn't matter.

Identifiers

  • The xml:id attribute is special.
  • It's value must be unique.
  • Identifies an element.
  • Facilitates linking from one element to another.

Identifiers

<root>
<item xml:id="i_123-a">This might be item 123a.</item>
<link ptr="#i_123-a">linking to it</link>
A second link to <link ptr="#i_123-a">123a</link>.
</root>

This is a well-formed document!

XML Elements

An XML element is a concept.

  • Start tag (or an empty tag)
  • Any attributes
  • Any content (including other elements)
  • End tag (unless the tag is empty)

Structure

  • A document has a root, sometimes called a document element.
  • It doesn't have to be named "root"
  • All other elements are descendants of the root.
  • All elements have exactly one parent.
  • Elements may have as many children as they need.

Best practises

  • Avoid stylistic tagging
  • Prefer semantic tagging
  • If something has meaning, mark it up.
  • Typographic conventions may not be important.

Best practises

  • Decide what you want from the markup. Set goals.
  • Document your editorial decisions.
  • Use tags that help acheive those goals.
  • Choose from small, fixed lists of values for types, renderings, etc.
