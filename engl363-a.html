XML Tags

  • Tags have names, and come in three flavours
  • Start and end with less-than and greater-than characters.
  • Start tag: 
    <tagname>
  • End tag: 
    </tagname>
  • Empty tag: 
    <tagname/>
  • It's all about the slashes

XML Tags

  • Names are case sensitive.
  • Names start with a letter.
  • Names may contain digits, hyphens, underscores, and colons.
  • Colons in names have a special purpose, so try to avoid them.

XML Tags

  • Well-formed: start tags must match up with an end tag.
  • Tags must be nested properly.

<title part='main'><b>Frankie & Johnny</b></title>

Nesting

Don't do this:


              Some text <i>in italic
              and <b>bold and</i> not italic</b>

Do do this instead:


              Some text <i>in italic and <b>bold
                and</b></i><b> not italic</b>

XML Attributes

Start and empty tags can have attributes. End tags do not.

Attributes have a name and a value.

<title part='main'></title>
<pagebreak n="27"/>
<title part='main' font='gothic'></title>

XML Attributes

  • Attribute names are case-sensitive.
  • Attribute names start with a letter
  • Attribute names may contain digits, hyphens, underscores, and colons.
  • Colons in attribute names have a special purpose, so try to avoid them.
  • Order doesn't matter.

              Frankenstein; or,
                the Modern Prometheus
            

              <title>Frankenstein; or,
                the Modern Prometheus</title>
            

              <title>Frankenstein</title>; or,
              <title>the Modern Prometheus</title>
            

              <title>
                <part type="main">Frankenstein</part>; or,
                <part type="sub">the Modern Prometheus</part>
              </title>

Header and Body

The TEI element must have two children


            <TEI xmlns="http://www.tei-c.org/ns/1.0">
              <teiHeader>
                ...
              </teiHeader>
              <text>
                ...
              </text>
            </TEI>

We'll work on the text element for now.

The Text

The text element can have three children


            <text>
              <front>
                Title page
                Table of contents
                Dedication
              </front>
              <body>
                The main content of the work
                Chapters, scenes, etc.
              </body>
              <back>
                Appendicies, Indexes
              </back>
            </text>

Best practises

  • Avoid stylistic tagging
  • Prefer semantic tagging
  • If something has meaning, mark it up.
  • Typographic conventions may not be important.

Best practises

  • Decide what you want from the markup. Set goals.
  • Document your editorial decisions.
  • Use tags that help acheive those goals.
  • Choose from small, fixed lists of values for types, renderings, etc.
